#  Requisitos Funcionais (RF)

Os requisitos funcionais descrevem **o que o sistema deve fazer** (funcionalidades e comportamentos).

###  RF01 – Divulgação Institucional

O sistema deve permitir a apresentação dinâmica de informação institucional:

- Missão
    
- Visão
    
- História
    
- Estrutura organizacional
    

---

###  RF02 – Gestão de Cursos

O sistema deve disponibilizar páginas dedicadas aos cursos:

- CTeSP
    
- Licenciaturas
    
- Pós-Graduações
    

Deve permitir:

- Listagem de cursos
    
- Visualização de detalhe de cada curso
    

---

###  RF03 – Gestão de Candidaturas Online

O sistema deve permitir:

- Submissão de candidaturas online
    
- Formulários dinâmicos
    
- Validação de dados
    
- Feedback ao utilizador após submissão
    

---

###  RF04 – Divulgação da Investigação

O sistema deve apresentar conteúdos relacionados com:

- CITECA
    
- Outras atividades de investigação
    

---

###  RF05 – Gestão de Eventos

O sistema deve permitir:

- Criação e divulgação de eventos atuais e futuros
    
- Listagem de eventos
    
- Página de detalhe de cada evento
    

---

###  RF06 – Gestão de Notícias

O sistema deve permitir:

- Publicação de notícias
    
- Listagem de notícias recentes
    
- Página individual de notícia
    

---

###  RF07 – Funcionalidade de Contactos

O sistema deve incluir:

- Formulário de pedido de informações
    
- Envio de mensagens para a instituição
    

---

###  RF08 – Backoffice Administrativo

O sistema deve incluir um backoffice que permita:

- Criar, ler, atualizar e eliminar (CRUD)
    
    - Cursos
        
    - Candidaturas
        
    - Eventos
        
    - Notícias
        
    - Conteúdos institucionais
        
    - Contactos
        

---

###  RF09 – Funcionalidade de Pesquisa

O sistema deve permitir:

- Pesquisa global por:
    
    - Cursos
        
    - Eventos
        
    - Notícias
        
    - Conteúdos institucionais
        

---

###  RF10 – Gestão de Erros

O sistema deve:

- Detetar erros (ex: submissão inválida)
    
- Informar o utilizador com mensagens claras
    
- Impedir submissão de dados inválidos
