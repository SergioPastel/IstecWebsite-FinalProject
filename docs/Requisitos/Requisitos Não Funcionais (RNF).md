#  Requisitos Não Funcionais (RNF)

Os requisitos não funcionais descrevem **como o sistema deve ser** (qualidade, desempenho, restrições técnicas, usabilidade, etc.).

---

###  RNF01 – Usabilidade

A plataforma deve possuir:

- Interface clara
    
- Navegação intuitiva
    
- Estrutura de conteúdos compreensível
    

---

###  RNF02 – Responsividade

O sistema deve ser responsivo e adaptar-se a:

- Desktop
    
- Tablet
    
- Smartphone
    

---

###  RNF03 – Conformidade com Brand Guidelines

O layout deve seguir:

- Manual de Identidade Visual do ISTEC Porto
    
- Cores, tipografia e elementos gráficos definidos
    

---

###  RNF04 – Público-Alvo e Internacionalização

O sistema deve:

- Ser adequado a utilizadores adultos
    
- Suportar idioma Português (PT)
    
- Suportar idioma Inglês (EN)
    

---

###  RNF05 – Experiência de Utilizador Fluida

A aplicação deve:

- Utilizar React.js no frontend
    
- Garantir navegação dinâmica (SPA-like experience)
    

---

###  RNF06 – Arquitetura Tecnológica

O sistema deve:

- Utilizar Laravel no backend
    
- Implementar separação clara entre frontend e backend
    
- Garantir organização adequada da lógica de negócio
    

---

###  RNF07 – Robustez e Tratamento de Erros

O sistema deve:

- Tratar falhas de submissão
    
- Garantir validação segura dos dados
    
- Evitar falhas críticas
    

---

###  RNF08 – Integridade e Validação de Dados

O sistema deve:

- Garantir validação tanto no frontend como no backend
    
- Assegurar consistência dos dados armazenados
